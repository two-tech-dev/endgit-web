# Performance & UX Improvement Plan

## Overview

Next.js 14 app (EndGit — plugin marketplace). Two categories of work: **bundle/runtime performance** and **UX polish**.

---

## 1. Performance — Bundle Size & Loading

### 1A. Replace framer-motion with CSS animations + lightweight IntersectionObserver hook

**Impact: HIGH** — framer-motion is ~30KB gzipped and is imported by `FadeIn`, `StaggerContainer`, `PluginCardGrid`, and every page that uses them.

- Create a `useInView` hook (lightweight, ~50 lines, no dependency)
- Create CSS-only `FadeIn` and `StaggerContainer` components using CSS `@keyframes` + the `useInView` hook to add a class when in viewport
- Update `PluginCardGrid` to use CSS transitions instead of `motion.a`
- Remove `framer-motion` from `package.json`

**Files:** `src/components/FadeIn.tsx`, `src/components/StaggerContainer.tsx`, `src/components/PluginCardGrid.tsx`, `src/components/HomeContent.tsx`, `package.json`

### 1B. Optimize react-syntax-highlighter imports

**Impact: MEDIUM** — Prism + all languages + vscDarkPlus theme is a large bundle (~200KB+).

- Switch to `react-syntax-highlighter/dist/esm/light` (lighter build)
- Only register the languages actually needed (e.g., `python`, `cpp`, `javascript`, `typescript`, `bash`, `json`)
- Use `AsyncLight` build or dynamic imports for the syntax highlighter

**Files:** `src/components/MarkdownTabs.tsx`

### 1C. Convert LatestPluginsSection to a server component

**Impact: MEDIUM** — Currently a client component that fetches via `useEffect`, meaning zero data on initial HTML, waterfall fetch, and unnecessary JS shipped.

- Remove `"use client"` from `LatestPluginsSection`
- Fetch data directly with `fetch()` (server-side) using `next: { revalidate: 60 }`
- Pass data as a server component rendered in `HomeContent` (or move the fetch into `page.tsx` and pass as prop)
- Remove client-side `useState`/`useEffect` for data fetching

**Files:** `src/components/LatestPluginsSection.tsx`, `src/components/HomeContent.tsx`, `src/app/page.tsx`

### 1D. Use Next.js Image component in PluginImage

**Impact: LOW-MEDIUM** — Currently uses raw `<img>` with manual `Image()` preloading. Bypasses Next.js image optimization (automatic WebP/AVIF, responsive srcset, lazy loading).

- Replace `<img>` with Next.js `<Image>` component
- Add the remote patterns already configured in `next.config.mjs`
- Use `onError` fallback instead of manual preloading

**Files:** `src/components/PluginImage.tsx`, `next.config.mjs` (already configured)

### 1E. Prefetch plugin detail pages on card hover

**Impact: LOW** — Navigation to plugin detail pages feels slow because the page isn't prefetched.

- Add `prefetch={true}` to plugin card `<Link>` components (Next.js default is true for visible links, but verify)
- Consider using `router.prefetch()` on hover for cards using `<motion.a>` (will be plain `<a>` after 1A)

**Files:** `src/components/PluginCardGrid.tsx`, `src/components/LatestPluginsSection.tsx`

### 1F. Deduplicate plugin data fetch on detail page

**Impact: LOW** — `generateMetadata` and the page component both call `getPlugin()`, causing two identical API requests.

- Use `React.cache()` or fetch once and pass through
- Or use Next.js `fetch` cache deduplication (automatic in Next.js 14 for same URL + same options)

**Files:** `src/app/plugins/[slug]/page.tsx`

## 2. Performance — Runtime & Perceived Speed

### 2A. Add Suspense boundaries for client components on plugin detail page

**Impact: MEDIUM** — `PluginAnalyticsChart`, `PluginRatings`, `DependencyGraph`, `VirusTotalCard` are all loaded via `dynamic()` but rendered synchronously in the component tree. Adding `<Suspense>` lets the page paint before these finish loading.

- Wrap each dynamic component in `<Suspense fallback={...}>`
- Move dynamic imports above the component or use proper streaming

**Files:** `src/app/plugins/[slug]/page.tsx`

### 2B. Reduce search debounce from 500ms to 300ms

**Impact: LOW** — 500ms feels sluggish. 300ms is standard for search-as-you-type.

**Files:** `src/components/PluginSearch.tsx`

### 2C. Inline critical CSS / preload Inter font

**Impact: LOW** — The Inter font is loaded via `next/font/google` which is already optimized. Verify no layout shift.

No changes needed — already handled by Next.js.

## 3. UX Improvements

### 3A. Terminal animation: don't replay on re-visit

**Impact: MEDIUM** — The typewriter animation replays every time the user navigates back to the home page. Store a `sessionStorage` flag to skip animation on subsequent visits and show the completed state immediately.

**Files:** `src/components/HomeContent.tsx` (TerminalMock)

### 3B. Add visible focus indicators for keyboard navigation

**Impact: MEDIUM** — Currently no `:focus-visible` styles. Users navigating with keyboard can't see which element is focused.

- Add `:focus-visible` outline styles to `.btn`, `.input`, `.card`, links, and interactive elements
- Use `outline: 2px solid var(--accent-primary)` with `outline-offset: 2px`

**Files:** `src/app/globals.css`

### 3C. Improve error states

**Impact: LOW-MEDIUM** — `LatestPluginsSection` silently returns `null` on error. `PluginAnalyticsChart` shows generic text. Show retry buttons and more informative messages.

**Files:** `src/components/LatestPluginsSection.tsx`, `src/components/PluginAnalyticsChart.tsx`

### 3D. Add loading="lazy" to below-fold images

**Impact: LOW** — Plugin card images already use `loading="lazy"` in PluginImage. Verify all images are optimized.

Already handled — no changes needed.

### 3E. Improve mobile touch targets

**Impact: LOW** — Pagination buttons and some interactive elements may be smaller than 44px on mobile.

- Audit pagination controls, sort options, filter buttons
- Ensure minimum 44px touch targets per WCAG guidelines

**Files:** `src/app/plugins/page.tsx`, `src/app/globals.css`

### 3F. Add smooth scroll-to-top on pagination

**Impact: LOW** — When clicking pagination, the page should scroll to the top of the plugin grid.

**Files:** `src/app/plugins/page.tsx` (add `scroll: true` to router.push or use anchor links)

## Git Workflow

- Create branch `refactor/performance-ux-improvements` from current branch before starting
- Run `pnpm run format && pnpm lint` before every commit
- Commit changes incrementally (one commit per logical group, e.g. "refactor: replace framer-motion with CSS animations")
- Do NOT touch unrelated files — only modify files listed in each step
- Push branch when complete

## Verification

- Run `pnpm build` to verify no build errors
- Run `pnpm lint` to verify no lint errors
- Run `pnpm type-check` to verify TypeScript
- Check bundle size with `npm run build` output (look at "First Load JS" per route)
- Manual test: home page, plugins list, plugin detail, mobile menu, theme toggle
